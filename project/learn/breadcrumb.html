วิธีทำเส้นทางนำทาง (Breadcrumb) พร้อม Schema ในธีม Blogger — คู่มือละเอียด

Breadcrumb คือเส้นทางเล็ก ๆ ที่บอกผู้อ่านว่ากำลังอยู่ตรงไหนของเว็บ และเป็นหนึ่งใน markup ที่ Google ชอบนำไปแสดงบนผลค้นหา บทความนี้สอนวิธีเพิ่ม breadcrumb พร้อม BreadcrumbList JSON-LD ในธีม Blogger ด้วย BlogKub แบบครบทุกขั้นตอน

อัปเดต 14 กรกฎาคม 2026 · อ่านประมาณ 9 นาที · ระดับ: เริ่มต้น–กลาง

สรุปสั้น: เพิ่มบล็อก เส้นทางนำทาง (Breadcrumb) ใน BlogKub ตั้งข้อความรายการแรก (เช่น “หน้าแรก”) แล้ว Export XML ไปอัปโหลด Blogger จากนั้น ใส่ป้ายกำกับ (Label) ให้ทุกโพสต์ เพราะ breadcrumb ใช้ป้ายกำกับแรกเป็นชื่อหมวด ระบบจะสร้างเส้นทาง หน้าแรก › หมวด › บทความ พร้อมฝัง BreadcrumbList schema ให้อัตโนมัติ มีสิทธิ์แสดงบนผลค้นหา Google

Breadcrumb คืออะไร

Breadcrumb (เส้นทางนำทาง) คือแถวลิงก์เล็ก ๆ ที่แสดงลำดับชั้นของหน้าปัจจุบันเทียบกับโครงสร้างเว็บ โดยทั่วไปอยู่เหนือหัวข้อบทความ มีหน้าตาแบบนี้:

หน้าแรก › ท่องเที่ยว › 10 ที่เที่ยวเชียงใหม่ที่ต้องไป

ชื่อ “breadcrumb” มาจากนิทานเรื่องการโปรยเศษขนมปังเพื่อจำทางกลับ เว็บใช้แนวคิดเดียวกัน คือให้ผู้อ่านรู้ว่ามาจากไหนและย้อนกลับไปหมวดหมู่หลักได้ในคลิกเดียว มันดูเรียบง่าย แต่ส่งผลทั้งด้านประสบการณ์ผู้ใช้และ SEO อย่างมีนัยสำคัญ

ทำไม Breadcrumb ถึงสำคัญต่อ SEO

  • Google นำไปแสดงแทน URL: ในผลค้นหา Google มักแทนที่ URL ยาว ๆ ด้วยเส้นทาง breadcrumb ที่อ่านง่าย เช่น blogkub.com › ท่องเที่ยว › เชียงใหม่ ทำให้ผลลัพธ์ดูเป็นระเบียบและน่าคลิกขึ้น
  • ช่วยบอทเข้าใจโครงสร้างเว็บ: breadcrumb เชื่อมโยงบทความกับหมวดหมู่ ทำให้ Google เข้าใจว่าเนื้อหาชิ้นนี้อยู่ในหัวข้อใหญ่อะไร ซึ่งช่วยการจัดกลุ่มและจัดอันดับตามหัวข้อ (topical authority)
  • เพิ่ม internal link ที่มีความหมาย: ลิงก์กลับหมวดหมู่ช่วยกระจายค่าน้ำหนักลิงก์ภายในเว็บและเปิดทางให้ผู้อ่านสำรวจเนื้อหาที่เกี่ยวข้องต่อ
  • ลดการกดออกทันที: เมื่อผู้อ่านเห็นทางไปหมวดที่สนใจ เขามีทางเลือกอื่นนอกจากกดปิด ทำให้อยู่ในเว็บนานขึ้น

ความแตกต่างระหว่าง Breadcrumb ที่ “แสดง” กับ “Schema”

Breadcrumb ที่ดีมีสองส่วนทำงานคู่กัน ส่วนแรกคือเส้นทางที่ ผู้อ่านมองเห็น บนหน้า อีกส่วนคือ BreadcrumbList JSON-LD ที่ซ่อนอยู่ในโค้ดสำหรับให้ Google อ่าน หลายเว็บทำแค่ส่วนที่มองเห็นแต่ลืม schema จึงไม่ได้เส้นทางสวย ๆ บนผลค้นหา จุดเด่นของ BlogKub คือสร้างทั้งสองส่วนให้ตรงกันโดยอัตโนมัติในทุกบทความ คุณจึงไม่ต้องกังวลว่าจะทำครบหรือไม่

วิธีเพิ่ม Breadcrumb ใน BlogKub ทีละขั้น

  1. เพิ่มบล็อกเส้นทางนำทาง เปิดคลังด้านซ้าย หมวด “บทความ & UX” แล้วลากบล็อก “เส้นทางนำทาง (Breadcrumb)” เข้ามา ระบบวางไว้ต้นบทความให้อัตโนมัติ
  2. ตั้งชื่อรายการแรก คลิกที่บล็อกเพื่อกำหนดข้อความรายการแรก เช่น “หน้าแรก” (ไทย) หรือ “Home” (อังกฤษ) ให้ตรงกับภาษาเว็บ
  3. Export XML และอัปโหลด กด Export แล้วนำไฟล์ไปที่ blogger.com → ธีม → เมนูสามจุด → กู้คืน/สำรอง → เลือกไฟล์ XML
  4. ใส่ป้ายกำกับให้ทุกโพสต์ ในตัวแก้ไขโพสต์ ช่อง “ป้ายกำกับ (Labels)” ให้ใส่หมวดของบทความ เช่น “ท่องเที่ยว” เพราะ breadcrumb ใช้ป้ายกำกับแรกเป็นชื่อหมวดในเส้นทาง
  5. ตรวจสอบด้วย Rich Results Test นำ URL บทความไปทดสอบที่ Google Rich Results Test เพื่อยืนยันว่า BreadcrumbList ถูกอ่านได้ ไม่มีข้อผิดพลาด

Breadcrumb ของ BlogKub วางอยู่ใน includable ของโพสต์ จึงมีข้อมูล data:post.labels อยู่ในขอบเขต ทำให้ดึงหมวดจากป้ายกำกับได้ถูกต้อง และเส้นทางที่แสดงจะตรงกับ schema เสมอ

เขียนและจัดหมวดอย่างไรให้ Breadcrumb มีประโยชน์สูงสุด

  • ใส่ป้ายกำกับที่เป็นหมวดจริง ๆ เช่น “ท่องเที่ยว”, “รีวิว”, “สุขภาพ” ไม่ใช่คำกระจัดกระจายอย่างชื่อสถานที่เฉพาะ ป้ายกำกับแรกควรเป็นหมวดที่กว้างพอจะรวมหลายบทความ
  • ใช้ระบบป้ายกำกับให้สม่ำเสมอ วางแผนหมวดหลักของเว็บไว้ล่วงหน้า (เช่น 5–8 หมวด) แล้วจัดทุกโพสต์ให้อยู่ในหมวดเหล่านี้ จะได้โครงสร้างเว็บที่ Google เข้าใจง่าย
  • อย่าใส่ป้ายกำกับมากเกินไป ป้ายกำกับแรกคือหมวดในเส้นทาง ถ้าใส่หลายป้ายควรจัดให้ป้ายหมวดหลักอยู่ลำดับที่เหมาะสม

ปัญหาที่พบบ่อยและวิธีแก้

เส้นทางไม่มีชื่อหมวด (เหลือแค่ หน้าแรก › บทความ)

เกิดเมื่อโพสต์ไม่มีป้ายกำกับ ให้เปิดโพสต์ในตัวแก้ไข Blogger แล้วเพิ่มป้ายกำกับหมวดในช่อง Labels จากนั้นบันทึกใหม่ เส้นทางจะสมบูรณ์ทันที

Rich Results Test แจ้งเตือน

โดยทั่วไป breadcrumb ของ BlogKub ผ่านการทดสอบ หากมีคำเตือนมักเป็นเพราะ URL หมวดของ Blogger ยังไม่มีบทความ ให้เผยแพร่บทความในหมวดนั้นอย่างน้อยหนึ่งชิ้นก่อน

Breadcrumb ไม่แสดงบนผลค้นหาทันที

Google ต้องเก็บข้อมูลหน้าใหม่ก่อน จึงอาจใช้เวลาหลายวันถึงสัปดาห์กว่าจะแสดงเส้นทางบนผลค้นหา สามารถเร่งได้ด้วยการส่ง URL ผ่าน Search Console

แนวทางปฏิบัติที่ดี (Best Practices)

  • ใช้ breadcrumb คู่กับโครงสร้างป้ายกำกับที่วางแผนดี จะได้ประโยชน์ทั้งการนำทางและ topical SEO
  • ตั้งชื่อรายการแรกให้ตรงกับภาษาเว็บและคงเส้นคงวา อย่าสลับไปมาระหว่าง “หน้าแรก” กับ “Home”
  • จับคู่กับสารบัญ (TOC)และบทความที่เกี่ยวข้อง เพื่อสร้างโครงข่ายลิงก์ภายในแบบ “ใยแมงมุม” ที่ช่วยทั้งผู้อ่านและการจัดอันดับ
  • ตรวจ Rich Results Test หลังอัปโหลดธีมครั้งแรก เพื่อยืนยันว่า schema ทำงาน

ตัวอย่างการวางโครงสร้างหมวดหมู่เว็บจริง

Breadcrumb จะทรงพลังก็ต่อเมื่อโครงสร้างหมวดหมู่ของเว็บถูกวางไว้ดี ลองดูตัวอย่างการวางหมวดสำหรับบล็อกสามแนวที่ต่างกัน เพื่อเป็นแนวทางออกแบบป้ายกำกับของคุณเอง

  • บล็อกท่องเที่ยว: หมวดหลักอาจเป็น “ในประเทศ”, “ต่างประเทศ”, “ทริปสั้น”, “คาเฟ่”, “ที่พัก” แต่ละบทความติดป้ายกำกับหมวดที่ตรงที่สุดหนึ่งหมวด เส้นทางจะกลายเป็น หน้าแรก › ในประเทศ › 3 วัน 2 คืนที่น่าน
  • บล็อกรีวิวสินค้า: จัดตามประเภทสินค้า เช่น “เครื่องใช้ไฟฟ้า”, “แกดเจ็ต”, “ของใช้ในบ้าน”, “ความงาม” เส้นทางแบบ หน้าแรก › แกดเจ็ต › รีวิวหูฟังไร้สายรุ่นใหม่ ช่วยให้ทั้งคนและ Google รู้ทันทีว่าเนื้อหาอยู่กลุ่มไหน
  • บล็อกสุขภาพ: แบ่งตามหัวข้อ เช่น “โภชนาการ”, “ออกกำลังกาย”, “สุขภาพจิต”, “โรคและอาการ” ซึ่งช่วยสร้างความเชี่ยวชาญเชิงหัวข้อ (topical authority) ที่ Google ให้ความสำคัญกับเนื้อหาสาย YMYL

หลักการคือเลือกหมวดหลักที่ “กว้างพอจะรวมหลายบทความ แต่แคบพอจะสื่อความ” ประมาณ 5–8 หมวดสำหรับบล็อกทั่วไปกำลังดี มากเกินไปจะกระจัดกระจาย น้อยเกินไปจะกว้างจนไม่มีความหมาย

Breadcrumb กับ E-E-A-T และความเชี่ยวชาญเชิงหัวข้อ

Google ประเมินเว็บด้วยแนวคิด E-E-A-T (ประสบการณ์ ความเชี่ยวชาญ ความน่าเชื่อถือ และความไว้วางใจ) และหนึ่งในสัญญาณของความเชี่ยวชาญคือการที่เว็บครอบคลุมหัวข้อใดหัวข้อหนึ่งอย่างเป็นระบบ Breadcrumb ที่จับคู่กับโครงสร้างหมวดหมู่ที่ดีช่วยแสดงให้เห็นว่าเว็บของคุณจัดกลุ่มเนื้อหาอย่างมีระเบียบ ไม่ใช่บทความกระจัดกระจายที่ไม่เชื่อมโยงกัน

เมื่อผู้อ่านหรือบอทคลิกจากบทความหนึ่งกลับไปหน้าหมวด แล้วเห็นบทความอื่น ๆ ในหัวข้อเดียวกันเรียงกันเป็นกลุ่ม นั่นคือหลักฐานเชิงโครงสร้างว่าเว็บคุณลงลึกในเรื่องนั้นจริง ยิ่งคุณเขียนบทความในหมวดเดียวกันมากและเชื่อมโยงกันดีผ่าน breadcrumb, บทความที่เกี่ยวข้อง และลิงก์ภายใน เว็บก็ยิ่งสะสม topical authority ซึ่งส่งผลต่อการจัดอันดับทั้งหมวดในระยะยาว

Breadcrumb บนมือถือและความเข้าใจผิดที่พบบ่อย

บนหน้าจอมือถือที่แคบ เส้นทางที่ยาวอาจตัดบรรทัดหรือดูอึดอัดได้ ทางแก้ที่ดีคือรักษาเส้นทางให้สั้นและมีความหมาย โดยทั่วไปสามระดับ (หน้าแรก › หมวด › บทความ) กำลังพอดี ไม่ควรซ้อนหมวดย่อยหลายชั้นจนเส้นทางยาวเกินไป เพราะทั้งเปลืองพื้นที่บนมือถือและทำให้ผู้อ่านสับสนมากกว่าช่วย

ความเข้าใจผิดที่พบบ่อยข้อแรกคือคิดว่า breadcrumb ต้องสะท้อนโครงสร้าง URL เป๊ะ ๆ ความจริงคือ breadcrumb สะท้อน “ลำดับชั้นเชิงตรรกะของเนื้อหา” ซึ่งอาจต่างจาก URL ก็ได้ เช่น URL ของ Blogger เป็นแบบวันที่ แต่ breadcrumb แสดงหมวดจากป้ายกำกับ ซึ่งมีประโยชน์ต่อผู้อ่านมากกว่า ความเข้าใจผิดข้อที่สองคือคิดว่าใส่ breadcrumb แล้วจะได้เส้นทางบนผลค้นหาทันที จริง ๆ ต้องรอ Google เก็บข้อมูลและประมวลผลก่อน ซึ่งใช้เวลา และข้อสุดท้ายคือการทำเฉพาะเส้นทางที่มองเห็นแต่ลืม schema ทำให้พลาดโอกาสแสดงผลแบบ rich result ไปอย่างน่าเสียดาย ซึ่งทั้งสามข้อนี้ BlogKub จัดการให้ถูกต้องตั้งแต่ต้น

สรุป

Breadcrumb เป็นองค์ประกอบเล็กที่ให้ผลใหญ่ มันช่วยให้ผู้อ่านไม่หลงทาง ช่วยให้ Google เข้าใจโครงสร้างเว็บ และทำให้ผลค้นหาของคุณดูเป็นระเบียบน่าคลิก ด้วย BlogKub คุณเพียงเพิ่มบล็อกเส้นทางนำทาง ตั้งชื่อรายการแรก และใส่ป้ายกำกับให้โพสต์อย่างมีวินัย ส่วน BreadcrumbList schema ระบบจะสร้างให้ตรงกับเส้นทางที่แสดงโดยอัตโนมัติ ทำให้เว็บ Blogger ของคุณพร้อมสำหรับ rich results ตั้งแต่วันแรก

คำถามที่พบบ่อย (FAQ)

Breadcrumb ใน BlogKub ฝัง Schema ให้อัตโนมัติไหม?

ฝังให้อัตโนมัติ ทุกบทความจะมี BreadcrumbList JSON-LD ที่ตรงกับเส้นทางที่แสดง ทำให้มีสิทธิ์ปรากฏเป็นเส้นทางนำทางบนผลค้นหา Google โดยคุณไม่ต้องเขียนโค้ดเอง

Breadcrumb ดึงชื่อหมวดจากไหน?

ดึงจากป้ายกำกับ (Label) แรกของโพสต์ ดังนั้นควรใส่ป้ายกำกับที่สื่อถึงหมวดหมู่ให้ทุกโพสต์ เพื่อให้เส้นทางเป็น หน้าแรก › หมวด › ชื่อบทความ อย่างถูกต้อง

ถ้าโพสต์ไม่มีป้ายกำกับจะเป็นอย่างไร?

เส้นทางจะเหลือเพียง หน้าแรก › ชื่อบทความ ซึ่งยังใช้ได้ แต่แนะนำให้ใส่ป้ายกำกับเสมอเพื่อเส้นทางที่สมบูรณ์และช่วยการจัดหมวดหมู่เว็บ

ช่วยจริง Google ใช้ breadcrumb แสดงลำดับชั้นเว็บบนผลค้นหาแทน URL ยาว ๆ ทำให้ผลค้นหาสวยและน่าคลิกขึ้น อีกทั้งช่วยผู้ใช้และบอทเข้าใจโครงสร้างเว็บและเชื่อมโยงหน้าเข้าด้วยกัน

Breadcrumb แสดงบนทุกหน้าหรือเฉพาะหน้าบทความ?

แสดงบนหน้าบทความเป็นหลัก เพราะเป็นจุดที่เส้นทาง หน้าแรก › หมวด › บทความ มีความหมาย ระบบของ BlogKub วาง breadcrumb ไว้ต้นบทความให้อัตโนมัติ

บทความในชุดคู่มือฟีเจอร์ BlogKub · เนื้อหาและ Schema (HowTo + BreadcrumbList + FAQPage) พร้อมเผยแพร่บน blogkub.com
