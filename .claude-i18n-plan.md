# English i18n rollout — progress checkpoint

Goal: an English version of every Thai page. Natural English prose, **no em dashes (—) and no middots (·)** anywhere in English content (titles, meta, schema names, body). Breadcrumb glyph `›` is fine.

## Infrastructure (DONE)
- `site/src/layouts/EnLayout.astro` — English site chrome (homepage look). Props `{ seo, thHref }`. `seo` is a raw head-HTML fragment (title, meta, og locale en_US, robots, JSON-LD). Import: `../../layouts/EnLayout.astro` for pages one level deep, `../../../layouts/EnLayout.astro` for `/en/blog/` and `/en/learn/` articles.
- `site/scripts/build-hreflang.mjs` (postbuild) — auto-injects reciprocal `th/en/x-default` hreflang into BOTH the TH page and its `/en/...` counterpart when both exist in dist. x-default → the EN URL. Skips pages that already declare x-default (hand-built homepages). No need to hand-edit TH pages.
- `site/scripts/build-en-learn-hub.mjs` (postbuild, runs first) — generates `dist/en/learn/index.html` from whatever `dist/en/learn/*.html` exist, so the hub never links to a missing article. Add articles and they appear automatically.
- postbuild order in `site/package.json`: en-learn-hub → hreflang → feeds → sitemaps.

## Pattern for an EN article page (learn/blog)
Mirror the TH `.astro`: same HTML structure in `body`, translate all Thai text to natural English. In `seo`, translate title/description/og, set `og:locale` en_US, `inLanguage:"en"`, canonical `https://www.blogkub.com/en/<path>`, Person schema name "Patpisan Dathong" alternateName ["PATPISAN DATHONG","Ben","LuMoo"], author url `/en/about`. Keep image URLs, dates (`2026-...T09:00:00+07:00`), and CSS classes identical. Convert `—`→comma/period, `·`→ `(...)`/comma/`&nbsp;`. Author box byline: `Patpisan Dathong (Ben)`, Pantip link text `Pantip (LuMoo)`. Internal links `/learn/x`→`/en/learn/x`, `/blog/x`→`/en/blog/x`, `/about`→`/en/about`. Keep `/builder.html` and `/builder` as-is. Pass `thHref="/<th-path>"`.
